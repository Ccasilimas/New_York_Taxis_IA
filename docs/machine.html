🤖 Análisis de Machine Learning: Predicción de Demanda

El objetivo de este proyecto es mejorar la eficiencia de la flota de taxis de Nueva York excluyendo el área del aeropuerto JFK. Mediante la optimización del uso de combustible y la identificación de zonas de alta demanda según la hora del día, día de la semana y proximidad a la ubicación actual, el proyecto se centra en mejorar la eficiencia operativa a través de una mejor asignación de vehículos, reduciendo costos al minimizar los tiempos de inactividad y mejorando la satisfacción del cliente con un servicio más confiable. Nuestro enfoque integra un modelo predictivo utilizando datos históricos, implementado a través de un backend basado en FastAPI y una interfaz web PHP fácil de usar. Esta integración respalda la toma de decisiones en tiempo real y los ajustes operativos para satisfacer eficazmente la demanda fluctuante de taxis.

Ejemplos de Uso

Identificación de Zonas de Alta Demanda

Usando PULocationID, la API FastAPI predice áreas con mayor demanda de taxis. Esto ayuda a los administradores a concentrar la flota en zonas específicas durante periodos de alta demanda.

Planificación Semanal

Analizando Day of Week, la API ajusta predicciones basadas en patrones semanales. Esto facilita la planificación y asignación de recursos, garantizando que haya suficientes taxis disponibles durante los días de mayor demanda.

Optimización Horaria

Usando Pickup Time, la API ajusta la disponibilidad de taxis según la hora del día. Esto permite ajustes dinámicos en la distribución de taxis para maximizar la eficiencia durante horas pico y reducir la flota en periodos de baja demanda.

Objetivos Alcanzados

Implementación FastAPI

Desarrollo de una API usando FastAPI para predicciones de demanda de taxis en varias zonas de Nueva York. Maneja solicitudes POST con datos específicos (ID de zona, día de la semana, hora de recogida) y devuelve predicciones de demanda.

Entrenamiento del Modelo de Regresión Lineal

Entrenamiento de un modelo de regresión lineal utilizando datos históricos de demanda de taxis, integrado con la API para predicciones en tiempo real.

Implementación Docker

Creación de Dockerfiles para aplicaciones FastAPI y PHP, asegurando un despliegue consistente y escalabilidad.

Interfaz Web PHP

Diseño e implementación de una interfaz PHP fácil de usar para interactuar con la API de predicción, incluyendo entrada de datos y visualización de resultados.

Descripción de Datos

  • ID de Ubicación de Recogida (PULocationID): Identifica zonas específicas de recogida
  • Día de la Semana: Números del 0 (Domingo) al 6 (Sábado)
  • Hora de Recogida: Formato 24 horas para análisis temporal
