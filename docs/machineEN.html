The objective of this project is to enhance the efficiency of New York City's taxi fleet while excluding the JFK Airport area. By optimizing fuel usage and identifying high-demand zones based on time of day, day of the week, and proximity to the current location, the project focuses on improving operational efficiency through better vehicle allocation, reducing costs by minimizing idle times, and enhancing customer satisfaction with more reliable service. Our approach integrates a predictive model using historical data, deployed via a FastAPI-based backend and a user-friendly PHP web interface. This integration supports real-time decision-making and operational adjustments to meet fluctuating taxi demand effectively.

Usage Examples

High Demand Zone Identification

Using PULocationID, the FastAPI API predicts areas with higher taxi demand. This helps administrators concentrate the fleet in specific zones during high-demand periods.

Weekly Planning

By analyzing Day of Week, the API adjusts predictions based on weekly patterns. This facilitates resource planning and allocation, ensuring sufficient taxis are available during peak demand days.

Time Optimization

Using Pickup Time, the API adjusts taxi availability according to time of day. This enables dynamic adjustments in taxi distribution to maximize efficiency during peak hours and reduce fleet size during low-demand periods.

Achieved Objectives

FastAPI Implementation

Developed an API using FastAPI for taxi demand predictions in various New York zones. Handles POST requests with specific data (zone ID, day of week, pickup time) and returns demand predictions.

Linear Regression Model Training

Trained a linear regression model using historical taxi demand data, integrated with the API for real-time predictions.

Docker Implementation

Created Dockerfiles for both FastAPI and PHP applications, ensuring consistent deployment and scalability.

PHP Web Interface

Designed and implemented a user-friendly PHP interface for interacting with the prediction API, including data input and result visualization.

Data Description

  • Pickup Location ID (PULocationID): Identifies specific pickup zones
  • Day of Week: Numbers from 0 (Sunday) to 6 (Saturday)
  • Pickup Time: 24-hour format for temporal analysis
